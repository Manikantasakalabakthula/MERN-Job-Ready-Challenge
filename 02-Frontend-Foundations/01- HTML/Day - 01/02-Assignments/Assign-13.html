1. What is DOM?

The Document Object Model (DOM) is a crucial concept in understanding how web pages are structured and manipulated.

Document Object Model

An API that outlines the logical structure of documents and how they are accessed and handled.

Programming Interface

Allows scripts (like JavaScript) to dynamically update the content, structure, and style of a document.

Tree Structure

The DOM represents a document as a logical tree where every branch ends in a node containing objects.

Node Types in the DOM Tree

  • Nodes: The generic term for any object in the DOM tree hierarchy.
  • Elements: Specific nodes representing HTML tags (e.g., <div>, <p>).
  • Attributes: Nodes containing information about elements (e.g., class, id).
  • Text Nodes: The actual text content residing inside an element.

2. HTML Tags

Tags are the building blocks of an HTML document, used to format and define the structure of content.

Opening Tag

Marks the beginning of an element. Enclosed in angle brackets. Example: <p>

Closing Tag

Marks the end of an element. Includes a forward slash. Example: </p>

Self-Closing Tag

Tags that do not require a separate closing tag because they cannot contain content. Example: <img>, <br>

Nested Tags

Placing HTML tags inside other HTML tags. Must be closed in the reverse order they were opened.

3. HTML Element

An individual component of an HTML document, encompassing the opening tag, attributes, content, and closing tag.

Relationships in the Element Hierarchy

  • Parent Element: An element that directly wraps another element.
  • Child Element: An element contained directly inside another element.
  • Sibling Element: Elements that share the same direct parent.
  • Empty Element: An element that cannot have any child nodes or text content (e.g., <input>).
<!-- Code Example demonstrating relationships -->
<div class="parent">
    <h1 class="sibling">Parent Title</h1>
    <p class="sibling">I am a child and a sibling!</p>
    <br /> <!-- Empty element -->
</div>

4. HTML Attribute

Attributes provide additional configuration or metadata details about an HTML element. They are always specified in the opening tag.

Global Attributes

Attributes common to all HTML elements, which can be safely used on any tag (e.g., title, style).

Element-Specific Attributes

Attributes restricted to certain tags (e.g., href for links, src for images).

id Attribute

A unique identifier across the entire document page. Used for styling or target anchoring.

class Attribute

A non-unique identifier used to group multiple elements together for shared CSS rules or scripts.

5. Structure of HTML Document

Every standard web page follows a structured skeletal framework to ensure proper rendering across different browsers.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Document Structure</title>
</head>
<body>
    <!-- Viewable web contents go here -->
</body>
</html>

6. <!DOCTYPE html>

An instruction to the web browser about what version of HTML the page is written in.

  • HTML5 Doctype: Declared simply as <!DOCTYPE html>. Case-insensitive but conventionally written in uppercase.
  • Standards Mode: Tells browsers to render the layout according to HTML5 and CSS specifications.
  • Quirks Mode: A legacy rendering mode triggered if the doctype is missing, causing the browser to emulate old buggy behaviors to avoid breaking archaic sites.

7. <html>

The Root Element of an HTML page. All other elements (except the doctype declaration) must be nested within this tag.

lang Attribute: Declares the primary language of the text in the document (e.g., <html lang="en">). It is critical for screen readers and SEO purposes.

8. <head>

A container for machine-readable information (metadata) about the document, which is not directly shown to the end user.

  • Metadata: Data providing information about other data (character sets, scripts, styles).
  • Resources: Links to external assets like stylesheets, scripts, and fonts.
  • SEO Information: Search Engine Optimization details that assist search crawlers in understanding your site.

9. <meta>

Defines metadata that cannot be represented by other HTML head elements. It is a self-closing element.

Attribute / Type Description / Usage
charset="UTF-8" Specifies the character encoding for the HTML document (covers almost all characters/symbols).
name="viewport" Controls layout on mobile viewports to ensure mobile responsiveness.
name="description" A short summary of the page content used by search engines on search result pages.
name="keywords" Comma-separated keywords relevant to the page (less weight in modern SEO).
name="author" Specifies the author or developer of the website document.

10. <title>

Defines the title of the document. This text is displayed in the browser's title bar or the page's tab.

  • Browser Title: Shown on the active tab header. Helps users navigate between open windows.
  • SEO Title: Serves as the primary clickable headline displayed in search engine result pages (SERPs).

11. Favicon

Short for "favorite icon". A small graphic image associated with a particular website, displayed in browser tabs, bookmarks, and history entries.

Implemented using the <link> element within the <head> block:

<link rel="icon" type="image/x-icon" href="/images/favicon.ico">
  • rel: Relationship attribute, set to "icon" or "shortcut icon".
  • type: Mime-type of the asset file (e.g., "image/png" or "image/x-icon").
  • href: The URL path specifying the location of the icon file.

12. Absolute Path

A full URL pointing to a specific location on the internet. It includes the protocol (e.g., HTTP/HTTPS) and domain name.

External Resource: Essential when referencing files or assets hosted outside your own server ecosystem.

<img src="https://example.com/assets/logo.png" alt="Absolute Path Example">

13. Relative Path

Points to a file relative to the current file position within the local directory folder structure.

Same Folder

Referenced directly by name or with a single dot.
src="photo.jpg" or src="./photo.jpg"

Parent Folder

Moving up one or more directory steps using double dots.
href="../index.html"

Child Folder

Moving down into a sub-directory layer.
src="images/logo.png"

14. Adding Text to the Body Tag

The <body> element contains all user-visible aspects of an HTML document.

  • Plain Text: Text dropped naked directly into the body without an enclosing structural tag will still render, but is considered poor practice because it lacks semantic description.
  • Browser Rendering: Browsers attempt to interpret text sequences, compressing multiples spaces or returns down into a single generic space unless instructed otherwise.

15. Comments in HTML

Comments are text entries added by developers to explain code logic. They are ignored completely by web browsers during execution.

<!-- This is a Single-Line Comment -->

  This is a 
  Multi-Line Comment

16. <p>

The Paragraph Element represents a structural block of plain context text text lines.

Block-Level Element: It automatically starts rendering on a brand new structural line layout block, and browsers append a default margin spacing before and after the element boundary blocks.

17. <pre>

The Preformatted Text element displays text exactly as written in the source code document.

  • Preserve Spaces: Multiple consecutive space characters are not collapsed.
  • Preserve Line Breaks: Carriage returns and line breaks appear exactly as typed.
  • Monospace Font: Typically renders out in a default uniform-width typography family (e.g., Courier).

18. Difference Between <p> and <pre>

Choosing between these components depends entirely on the formatting criteria of your raw content payload data.

Feature Paragraph Tag (<p>) Preformatted Tag (<pre>)
Space Handling Collapses multiple blank spaces into one. Retains and renders every single space.
Line Break Handling Ignores source returns (requires <br>). Renders explicit line breaks directly.
Text Wrapping Automatically wraps text to fit container. Does not wrap text unless explicitly configured.
Default Font Family Standard proportional font (sans-serif/serif). Fixed-width Monospace font layout.

19. Adding Styles to Paragraphs

Visual custom presentation characteristics can be added directly via HTML features using the style attribute framework.

Inline Style: Declaring CSS attributes right inside the element tag marker. Ideal for quick one-off design exceptions.

<p style="color: blue; font-size: 16px;">Styled Paragraph Text</p>

20. Break Tag (<br>)

Creates a simple line break layout step within structural text flow blocks.

  • Empty Element: It has no structural text content inside and does not take an ending close wrapper tag marker.
  • Line Break: Forces subsequent inline content text properties onto an immediate new row line layout without opening a fresh margins structural block.

21. HTML Entities

Used to represent reserved HTML characters, hidden whitespaces, or symbols missing from standard character keyboard sets.

  • Reserved Characters: Symbols like < and > must be written as &lt; and &gt; so browsers do not misinterpret them as true structural tag hooks.
  • Non-Breaking Space (&nbsp;): Creates a clean space that will not automatically collapse or break into a separate new wrapping line layout row.
  • Special Symbols: Used to render copyrights (&copy; → ©), currency indicators, or Greek mathematical letters.
